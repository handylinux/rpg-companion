# Кириллица в коде — совместимость со старой БД

Этот файл — единственное место, где задокументированы все оставшиеся кириллические строки в JS/TS коде.
Они существуют **не как UI-строки**, а как ключи/значения для совместимости со старой схемой БД.
Когда БД будет мигрирована — каждый пункт здесь должен быть удалён.

---

## 1. `components/screens/WeaponsAndArmorScreen/WeaponModificationModal.js`

### 1.1 `translateModTokenToRu` — словарь перевода префиксов модов
```js
// строки 73–95
const map = {
  Hardened: 'Укреплённый',
  Improved: 'Улучшенный',
  // ... и т.д.
};
```
**Причина:** префиксы модов в БД хранятся на английском, но `range_name` и имена оружия — на русском.
Этот маппинг нужен, пока БД не переведена на единую схему с английскими ключами.
**Что сделать при миграции БД:** удалить функцию `translateModTokenToRu` целиком, брать локализованное имя из i18n по английскому ключу.

### 1.2 `rangeOrder` — порядок дистанций
```js
// строка 173
const rangeOrder = ['Близкая', 'Средняя', 'Дальняя', 'Экстремальная'];
const currentRangeName = String(baseWeapon.range_name ?? 'Близкая').trim();
```
**Причина:** поле `range_name` в БД хранится по-русски. Массив используется для вычисления сдвига дистанции при применении эффектов модов.
**Что сделать при миграции БД:** заменить на английские ключи (`['Close', 'Medium', 'Long', 'Extreme']`), `range_name` в БД тоже мигрировать.

### 1.3 Фолбэки `'Близкая'` в JSX
```js
// строки 374, 418
weapon?.range_name ?? 'Близкая'
modifiedWeapon.range_name || 'Близкая'
```
**Причина:** те же данные из БД. Фолбэк совпадает с дефолтным значением в БД.
**Что сделать при миграции БД:** заменить на `tWeaponsAndArmorScreen('weapon.rangeDefault')` или убрать фолбэк если поле станет обязательным.

## 3. `i18n/equipmentNormalizer.js`

### 3.1 `AREA_LABELS_RU` — маппинг зон брони
```js
// строки 2–6
const AREA_LABELS_RU = {
  Head: 'Голова',
  Body: 'Тело',
  Hand: 'Рука',
  Leg: 'Нога',
};
// строка 10–11
if (labels.includes('Рука') && labels.includes('Нога') && labels.includes('Тело'))
  return 'Руки, Ноги, Тело';
```
**Причина:** зоны защиты брони в БД хранятся на английском, но нормализатор сразу переводит их в русские строки для отображения.
**Что сделать при миграции:** убрать `AREA_LABELS_RU`, хранить английские ключи, переводить через i18n на уровне UI.

### 3.2 Ключ `Название`
```js
// строка 19
Название: piece.name,
```
**Причина:** старая схема объектов снаряжения использовала русский ключ `Название`.
**Что сделать при миграции:** убрать, везде использовать только `name`.

## 7. `data/traits/traits.json`

### 7.1 Поле `cyrillicName` — кириллическое имя черты
```json
"cyrillicName": "Принудительная эволюция"
```
**Причина:** в БД черты хранятся по кириллическому имени (поле `name` в объекте черты персонажа). `cyrillicName` нужен для сопоставления записей из БД с новыми `id`-ключами.
**Что сделать при миграции БД:** мигрировать поле `name` черты в БД на латинский `id`, удалить `cyrillicName` из `traits.json`.

### 7.2 Поле `forcedSkills` — кириллические названия навыков
```json
"forcedSkills": ["Энергооружие", "Наука", "Ремонт"]
```
**Причина:** навыки в БД и в `ALL_SKILLS` хранятся по кириллическим именам. `forcedSkills` используется для сопоставления с навыками персонажа.
**Что сделать при миграции:** мигрировать ключи навыков на латинские `id`, заменить кириллические строки на `id` вида `"energy_weapons"`, `"science"`, `"repair"`.

### 7.3 Поле `skillModifiers` — кириллические ключи навыков
```json
"skillModifiers": { "Выживание": 2 }
```
**Причина:** та же совместимость с кириллическими ключами навыков в БД.
**Что сделать при миграции:** заменить на латинские `id` навыков.

## Порядок очистки при миграции БД

1. Мигрировать поля в БД: `range_name` → английские значения, убрать русские ключи объектов
2. Удалить `translateModTokenToRu` в `WeaponModificationModal.js`, добавить i18n-ключи для префиксов модов
3. Обновить `rangeOrder` и фолбэки `'Близкая'`
4. Убрать кириллические ключи из `modsEquip.js`
5. Рефакторить `equipmentNormalizer.js` — перенести перевод зон в i18n
6. Убрать ключ `Название` из `equipmentNormalizer.js` и `equipmentCatalog.js`
7. Исправить фолбэк в `appI18n.js`
8. Удалить этот файл
