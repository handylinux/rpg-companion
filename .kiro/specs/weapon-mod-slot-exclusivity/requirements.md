# Requirements Document

## Introduction

В системе модификации оружия необходимо обеспечить эксклюзивность слотов: на оружие можно установить только одну модификацию из каждой категории (ресивер, ствол, магазин, ложе и т.д.). При установке новой модификации из той же категории, что уже установлена, старая модификация автоматически удаляется вместе со всеми своими бонусами, и применяется новая. Характеристики оружия всегда должны рассчитываться от базовых значений с учётом только актуально установленных модификаций.

## Requirements

### Requirement 1

**User Story:** Как игрок, я хочу устанавливать модификации на оружие по одной из каждой категории, чтобы не накапливать бонусы от нескольких модификаций одного типа.

#### Acceptance Criteria

1. WHEN игрок устанавливает модификацию категории X на оружие THEN система SHALL разрешить установку только одной модификации из категории X одновременно.
2. WHEN игрок устанавливает вторую модификацию из той же категории X THEN система SHALL автоматически удалить ранее установленную модификацию из категории X.
3. WHEN старая модификация удаляется THEN система SHALL полностью отменить все её бонусы к характеристикам оружия (урон, скорострельность, дистанция, вес, цена, эффекты, качества).
4. WHEN новая модификация устанавливается взамен старой THEN система SHALL применить только бонусы новой модификации к базовым характеристикам оружия.

### Requirement 2

**User Story:** Как игрок, я хочу видеть корректные характеристики оружия после замены модификации, чтобы понимать реальные параметры своего оружия.

#### Acceptance Criteria

1. WHEN игрок заменяет ресивер с +1 урона на ресивер с +3 урона THEN система SHALL показать урон = базовый урон + 3 (а не базовый + 1 + 3).
2. WHEN игрок открывает модальное окно модификаций для уже модифицированного оружия THEN система SHALL отображать текущие установленные модификации как выбранные.
3. WHEN игрок применяет изменения THEN система SHALL пересчитать все характеристики оружия от базовых значений с учётом всех актуальных модификаций.
4. IF оружие имеет установленные модификации из нескольких категорий THEN система SHALL корректно применить все их бонусы без дублирования.

### Requirement 3

**User Story:** Как игрок, я хочу хранить информацию об установленных модификациях на оружии, чтобы при повторном открытии модального окна видеть текущее состояние.

#### Acceptance Criteria

1. WHEN модификация устанавливается на оружие THEN система SHALL сохранить запись об установленной модификации с указанием категории и названия мода.
2. WHEN игрок открывает модальное окно модификаций THEN система SHALL загрузить и отобразить ранее установленные модификации как уже выбранные.
3. WHEN игрок сохраняет персонажа THEN система SHALL сохранить информацию об установленных модификациях вместе с данными персонажа.
4. WHEN игрок загружает персонажа THEN система SHALL восстановить все установленные модификации на оружии.

### Requirement 4

**User Story:** Как игрок, я хочу иметь возможность установить модификации из разных категорий одновременно, чтобы полностью кастомизировать оружие.

#### Acceptance Criteria

1. WHEN игрок устанавливает модификации из разных категорий (например, ресивер и ствол) THEN система SHALL применить бонусы от всех установленных модификаций.
2. WHEN игрок заменяет одну из модификаций THEN система SHALL сохранить модификации из других категорий нетронутыми.
3. IF оружие имеет доступные слоты для нескольких категорий THEN система SHALL позволить установить по одной модификации в каждый слот независимо.
